Web UI Framework Reference

The Web UI framework provides tools, such as widgets, events, effects, and animations, for Web application development. You can leverage these tools by just selecting the required screen elements and creating applications.

The Web UI service is based on a template and works on a Web browser, which runs on the WebKit engine. You can code Web applications using the Web UI, standard HTML5, and Tizen APIs. You can also use different widgets with CSS animations and rendering optimized for Tizen Web browsers.

For more information about the basic structure of a page in the Web application using the Web UI service, see Application Page Structure.

Framework Services

The Web UI framework consists of the following services:

  • Web widgets

    Tizen widgets, such as control bar and virtual list, and jQuery Mobile widgets are available.

  • Web themes

    CSS themes and resources for Tizen Web widgets (compatible with the widget source code) are available.

  • Loader

    Configurations, such as Web theme and internationalization set-up, can be loaded.

The Web UI framework services are supported by the following JavaScript projects:

  • jQuery version 1.8.2

    jQuery is a JavaScript library used for selecting HTML elements, traversing through HTML pages, and handling events.

  • jQuery Mobile version 1.2.0

    jQuery Mobile is an HTML5-based mobile Web framework for responding to touch events.

  • Globalize version 0.1.0a2

    Globalize is a JavaScript library that supports globalization and localization.

Limitations

The Web UI framework has the following limitations:

  • The framework runs only on browsers supporting the HTML5/CSS standards. The draft version of the W3C specification is not fully supported.
  • The Tizen Web widgets are optimized for 1280 x 720 (720p) display resolution target screen with high-definition (HD) dpi. The widgets are scaled down for lower display resolution targets.
  • Resource files per resolution (High-Medium-Low) are currently not supported.
  • Browser-specific zoom options, such as pinch zoom and smart zoom, are not supported for scaling titles and tool bars.
  • The jQM swatch feature is not supported. Therefore, the layout is broken if you change the default data-theme value and the data-mini attribute is not supported.
