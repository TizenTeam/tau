Application Page Structure

In the Tizen Web UI framework, the application page structure is based on jQuery Mobile as described in the jQuery Mobile documentation for the anatomy of a page.

As shown in the following figure, a Tizen page consists of 3 elements: header, content, and footer.

Figure: Tizen page elements

The page and its elements (header, content, and footer) are all <div> blocks with a specific data-role property. The header is placed at the top, and displays the page title. The content is the area below the header, showing the main content of the page. The footer is at the bottom, and contains the page menu.

To add a Tizen page to the application, use the following code:

<div data-role="page" data-add-back-btn="footer">

<!--Page area-->
   <div data-role="header"><!--Header area--></div>
   <div data-role="content"><!--Content area--></div>
   <div data-role="footer"><!--Footer area--></div>
</div>

The user cannot select any content in the default header and footer, and the default header and footer also do not fire context menu events.

The following table describes the page options.

Option Input type Description
data-add-back-btn String Sets the page back button and its location.

The available values are true and false, and the default value is false.

The Tizen page can define callbacks for events as described in the jQueryMobile documentation for events.

You can use the refresh method with the Tizen page to update the current page:

<div data-role="page"></div>

$("#.selector").page('refresh');
Note
The refresh method is supported since Tizen 2.0.
