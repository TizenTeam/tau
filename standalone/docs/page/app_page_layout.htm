Application Page

Page Layout

In the Gear Web UI framework, the application page structure is based on header, content elements

The header is placed at the top, and displays the page title. The content is the area below the header, showing the main content of the page.

To add a page to the application, use the following code:

<!--Page Layout-->

<div class="ui-page">
 <header class="ui-header"></header>
 <div class="ui-content"></div>
 <footer class="ui-footer"></footer>
</div>

<!--Page Layout with more button in header-->
<div class="ui-page">
  <header class="ui-header ui-has-more">
   <h2 class="ui-title">Call menu</h2>
   <button type="button" class="ui-more ui-icon-overflow">More Options</button>
  </header>
  <div class="ui-content">
	Content message
  </div>
  <footer class="ui-footer">
   <button type="button" class="ui-btn">Footer Button</button>
  </footer>
</div>

To find the current active page, use the 'ui-page-active' css class.

Page Interface

Interface Description
gear.ui.changePage ( to ) Programmatically change to another page. argument 'to' is a page object or string.
gear.ui.back() This method loads the previous page in the history list.

window.addEventListener( 'tizenhwkey', function( ev ) {
	if( ev.keyName == "back" ) {
		/* Call window.history.back() to go to previous browser window */

	}
}
