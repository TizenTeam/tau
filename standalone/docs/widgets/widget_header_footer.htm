Header and Footer

The header and footer widget is used to set the default bar on the page. They are closely connected with the page winset because most options are defined in the page. The header and footer widget styles are defined in the HTML markup of the page.

Note
The default button style (data-role="button") is not supported in the header. You must use the <a> tag to add a button.

To add a header widget to the application, use the following code:

<div id="foo" data-role="header" data-position="fixed">
   <h1>NBeat UI</h1>
   <a>Button 1</a>
</div>

$("#foo").barlayout('show');

The following list describes header styles:

Table: Header styles
Header style HTML code
Title with 1 button 
<div data-role="header" data-position="fixed">
   <h1>NBeat UI</h1>
   <a>Button</a>
</div>
Title with 2 buttons 
<div data-role="header" data-position="fixed">
   <h1>NBeat UI</h1>
   <a>Button 1</a>
   <a>Button 2</a>
</div>
Title with an icon and button 
<div data-role="header" data-position="fixed">
   <img src="icon-tizen.png" />
   <h1>Icon Title</h1>
   <a>text1</a>
</div>
Title with subtext(multiline) 
<div data-role="header" data-position="fixed">
   <h1>Title Text multiline</h1>
   <span class="ui-title-text-sub">tizen@samsung.com</span>
</div>
Title with subtext and image (multiline) 
<div data-role="header" data-position="fixed">
   <img src="icon-tizen.png"/>
   <h1>Title demo</h1>
   <span class="ui-title-text-sub">Title image multiline 2 Buttons</span>
   <a data-icon="naviframe-more"></a>
   <a data-icon="naviframe-edit"></a>
</div>

The header and footer can define callbacks for events as described in the jQueryMobile documentation for events.
